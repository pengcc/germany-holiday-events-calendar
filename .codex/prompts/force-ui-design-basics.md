# Force UI Design Basics

Use `ui-design-basics` as supporting guidance inside the current primary workflow.

Inspect the existing UI system, components, tokens, styles, and project conventions first. Define
one clear page, screen, or flow goal before proposing UI direction.

Keep the result quiet, clean, practical, readable, and minimally polished. Reuse established
components and semantic styling, and preserve relevant UI states and accessibility basics.

Do not install or invoke unapproved skills, MCP servers, CLIs, dependencies, or tooling. Return to
`plan-with-context`, `execute-plan`, `code-review`, or `project-architecture-plan` for the primary
workflow boundary.
