# UI Quality

## Visual Direction

Use a clean, modern travel-planning tool aesthetic. The product should feel light, readable, and useful for families comparing dates, not like an admin dashboard or a generic calendar plugin.

## UI System

- Use shadcn/ui conventions with Tailwind CSS.
- Use Radix primitives for accessible interactive controls.
- Use lucide-react icons for familiar actions.
- Prefer custom calendar/heatmap UI for the core comparison experience.

## Interaction Requirements

- State multi-select should be clear and keyboard-operable.
- Calendar/heatmap meaning must be supported by labels or legends, not color alone.
- Desktop and mobile layouts must be checked for text fit and non-overlap.
- Avoid decorative one-note palettes and generic dashboard cards.
