# Holiday Sync Germany Execute Plan

Use this skill when executing an approved or saved plan.

## Steps

1. Read the saved plan in `dev_locals/plans/`.
2. Re-check Git status before editing.
3. Keep edits scoped to the plan.
4. Avoid product scope creep, especially backend runtime, personal data, telemetry, or paid services.
5. Run relevant checks.
6. Self-review changed files before finishing.
7. Make a checkpoint commit after a coherent work unit if checks pass and Git is available.

## Reporting

Report changed files, checks run, checks skipped, and any follow-up risks.
